# AI Module Conventions (Robin / Genkit)

## Architecture
- **Framework**: Genkit with `@genkit-ai/google-genai` plugin
- **Model**: `gemini-2.5-flash` via `googleAI.model()`, exported as `geminiFlash` from `@/ai/genkit`
- **Singleton**: `ai` and `geminiFlash` are imported from `@/ai/genkit` everywhere
- **Fallback**: When API key is missing, `ai` is a stub returning safe user-facing error messages

## Flow Pattern
- Every flow file starts with `'use server'` (Next.js Server Actions)
- Import `{ ai, geminiFlash }` from `@/ai/genkit` and `{ z }` from `zod`
- Use `ai.generate()` with `{ model: geminiFlash!, prompt, output: { schema }, config: { temperature: 0.05 } }`
- Safety settings: `BLOCK_NONE` for all harm categories — Robin moderates via system prompt
- Always wrap `ai.generate()` in try/catch with specific checks for 429/quota/RESOURCE_EXHAUSTED errors
- Include regex-based extraction as fallback when AI fails

## Tool Pattern
- Define with `ai.defineTool({ name, description, inputSchema, outputSchema }, handlerFn)`
- Import `{ z }` from `'genkit'` (not `'zod'`) for tool schemas
- Use Firestore Admin SDK (`getAdminDb()` from `@/firebase/admin`), not client SDK

## Key Types (from `@/lib/types`)
- `Player`: id, firstName, lastName, phone, email, avatarUrl, ownerId
- `Court`: id, name, location, address, ownerId, timezone
- `Group`: id, name, members[], ownerId
- `ChatInput` / `ChatOutput`: Zod schemas for Robin conversation I/O
- `SmsIntent`: 'accept' | 'decline' | 'cancel' | 'scheduling' | 'question' | 'unknown'

## Error Handling
- Retry once on rate limit (429/quota/RESOURCE_EXHAUSTED) with 10s delay
- Fall back to regex extraction if retry fails — never show raw errors to users
- Check for SAFETY/blocked errors separately and redirect to scheduling

## Environment
- API key: `GOOGLE_GENAI_API_KEY` (env var and Firebase Secret Manager)
- Timezone: `America/New_York`
- Deployed via Firebase App Hosting (push to `main` triggers build)
